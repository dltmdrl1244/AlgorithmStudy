# 문제
링크 : [1034 램프](https://www.acmicpc.net/problem/1034)

해결여부 : No

# 문제 풀이
> "특정 행을 켤 수 있는 조합으로 다른 행도 켤 수 있는지 탐색"

한 문장으로 요약하자면 이렇게 가능.

- i번째 행을 켜는 방법 I는, 많은 경우의 수들 중 딱 한 가지 (꺼진 것만 모두 켜는 경우)
- 그리고 I로, i번째 행과 초기 상태가 똑같이 생긴 행도 켤 수 있다.
- 이 때 I에 필요한 수가 스위치를 켤 수 있는 횟수 k보다 작으면 안 되고, 또 I-k가 홀수이면 안 됨
- 짝수인 경우에는 특정 스위치를 껐다 켰다 반복하는 것으로 횟수를 소모할 수 있지만, 홀수인 경우에는 반드시 하나 이상의 스위치의 상태를 바꿔야 하기 때문
- 각 행에 대해서 I를 구하고, 마침 I가 다른 행도 켤 수 있으면 그것을 함께 탐색

스위치를 켤수록 각각의 행에 대해 상태가 뒤죽박죽이 되니까 '켤 스위치'를 조합을 이용해서 브루트 포스를 떠올렸지만 너무 터무니없이 큰 횟수가 나와서 다른 방법을 사용

'특정한 케이스를 만족하는' 경우를 먼저 찾고 -> 이것으로 동시에 해결 가능한 것을 찾는다.

여러 열쇠구멍이 있고 열쇠를 만들 수 있을 때, 가장 많은 구멍을 열 수 있는 열쇠는 몇 개를 열 수 있나? 를 생각하면,
- 당연히 1번 열쇠구멍에 맞는 열쇠를 만들고 이걸로 다른 구멍에 넣어보기를 반복하지,
- 만들 수 있는 모든 경우의 수로 열쇠를 왕창 만들고 이걸로 모든 구멍에 넣어보진 않는다.

생각이 잘 떠오르지 않아서 결국 답을 검색해 보았는데 발상의 전환이 필요함을 알게 되었다.
여담으로 이러한 매커니즘을 'ad hoc' 이라고 부른다고 함
