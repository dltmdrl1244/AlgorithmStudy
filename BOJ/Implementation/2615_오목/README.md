# 문제

링크 : [2615 오목](https://www.acmicpc.net/problem/2615)

해결여부 : YES

# 문제 풀이

오목 방향은 →, ↓, ↗, ↘ 으로 탐색
전체 바둑판을 행렬 중 행 우선으로 탐색하므로, 우상향 대각선의 경우 시작점보다 끝점을 먼저 탐색을 하게 된다. 이 부분을 보완해 주기 위해서 시작점에서 탐색할 방향의 반대 방향으로 1칸 뒤를 비교하여 시작점과 같은 돌이 있으면 이 시작점은 전체 덩어리의 시작점이 아니므로, 탐색을 진행하지 않는다. (나중에 시작점에서 하면 됨)

방향 탐색 여부는 비트 마스킹을 사용한다. 예컨대 `[i][j]` 칸이 비트 마스킹(visited) 값이 `1100` 이라면 '이 칸은 상하, 좌우 방향으로는 탐색을 했다', 즉 이 방향으로는 오목을 이루지 않는다 라는 것을 의미.

동시에 여러 군데에서 오목이 발생하거나, 흑돌 백돌이 동시에 오목이 발생하지 않는다고 했으므로, 오목을 발견하자마자 색깔과 좌표를 출력하고 코드를 종료한다.
